| Category                  | Details                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Profile**               | Early-stage or growing teams<br>Limited data engineering resources<br>Mix of operational and analytical workloads<br>Often starting with a single database                                                                                                                                                                                                                                                                                                           |
| **Common Starting Point** | PostgreSQL (or similar OLTP system)<br>Ad hoc reporting or BI directly against the database                                                                                                                                                                                                                                                                                                                                                                          |
| **Key Challenge**         | Simplicity vs. scalability<br>Avoiding premature complexity while still preparing for growth                                                                                                                                                                                                                                                                                                                                                                         |
| **Recommended Approach**  | **Keep OLTP Simple**<br>Use PostgreSQL (managed on AWS) for core operations<br>Avoid over-engineering early<br><br>**Introduce Object Storage Early**<br>Land periodic extracts into Amazon S3<br>Start building a historical record outside the database<br><br>**Use Lightweight Analytics**<br>Query data using DuckDB or Amazon Athena<br>Avoid standing up heavy infrastructure<br><br>**Start a Basic Catalog**<br>Use AWS Glue Data Catalog to track datasets |
| **Outcome**               | Minimal overhead<br>Early separation of OLTP and analytics<br>Foundation for future scale                                                                                                                                                                                                                                                                                                                                                                            |
